please.js is a promise-resolution tool. It's intention is to provide syntax sugar for promise resolution since I find the vanilla js promises syntax to be a bit clunky. please.js uses a function chain syntax that attempts to have a intuitive flow to it.

Github Link

The example below is how lite.js uses please.js on page load. The views often load content (usually html or text) and bind it to the page, as well as load data (usually json) and bind it to the content. As such, it wants to make sure the content and data have both been loaded succcessfully before attempting the binding operations.

example usage

                
please.do(_loadContent())
    .and(_loadData())
    .then((x, y)=>{
        _bindContent(_lite.content);
        _bindData(_lite.data);
    });
